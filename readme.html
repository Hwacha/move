F-Painter

  1. Description
  2. How to Use
  3. What to Add

Description

  • F-Painter is a JavaScript web utility that visualizes 3-place functions as animated colored tiles.
  • The user enters in functions r, g, b with respect to x, y, and t which correspond to a color component value at position (x, y) and at time t.
  • The user can play, pause, and stop the animated graph.

How to Use

  1. Type functions in for R, G, B.
    Possible values: —
    • numbers: integer value e.g. 500 or decimal value e.g. 1.45
    • variables: x, y, t
      NOTE: no other variables will be recognized.
    • infix operators:
      • + (addition)
      • - (subtraction)
      • * (multiplication)
      • / (division)
      • mod (modulus)
      • ^ (exponentiation)
      • = (equals) Note that relations evaluate to 1 if the relation holds, and 0 if the relation does not hold.
      • > (greater than)
      • < (less than)
      ex. (1 + x), (t ^ 4), ((x * y) % t)
      NOTE: you need parenthesis around any expression except a number or variable
  2. Press ENTER or click "apply".
  3. Start and stop
    • Click the play button to begin the animation.
    • Click pause to pause the animation. Pressing play will continue from where it was paused.
    • Click the stop button to stop the animation and reset to the first frame.
      HINT: If no function depends on t, then using stop is the same as play: and performs better!

What to Add

  • Syntax
    • parentheses
    • constants like e and π
    • one argument operators like trig functions, log and square root
  • Performance
    • Remove object wrapper for numbers, variables, and maybe functions and instead just use closures. This will be less readable, but will probably allow for a larger frame and smaller tiles.
    • Find a better way to timeout the animation to avoid crashes. Right now it resets the time counter when it reaches 256^2.
    • Automatically stop time interval if no function is dependent on t.
    • There are some issues with the creation of some functions that should be valid.
  • Conceptual

Right now it's an aesthetic choice, so that I can avoid to transform the tile structure (like constantly rotating) so the user can see the whole color space. I think it would be cool to implement a parsing scheme which could receive and display n-place functions in (n + 1)-dimensional space.
      Add user customization of frame size, tile size, etc.
      Add real-time user interaction with camera frame and maybe controllable avatar tiles.
